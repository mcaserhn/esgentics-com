# ESGentics.com 项目复盘：经验与原则

## 项目概述

ESGentics.com 是一个面向下一代建筑技术公司的网站，利用AI和NVIDIA硬件提供智能建筑解决方案。项目从前端开发、后端实现到部署上线，形成了一套完整的开发流程和方法论。

**项目进展**：
- 已完成核心功能开发和Azure App Service部署
- 已更新办公地点信息（新加坡和巴黎）
- 已完善项目文档和交接资料
- 已配置GitHub Actions自动化部署流程
先通过QWEN通过提示词输出了完整的网站设计文档md，再用TRAE先规划网站开发计划并形成文档md，最后完成网站发布

## 一、技术选型与架构设计

### 1. 技术栈选择原则

**核心原则**：
- **现代化与稳定性平衡**：选择 Next.js 14 这样的现代框架，同时确保核心依赖的稳定性
- **全栈一致性**：使用同一技术生态（React + Next.js），减少技术栈复杂度
- **可扩展性**：选择支持模块化和组件化的技术，便于未来功能扩展
- **性能优先**：考虑框架的性能特性，如 Next.js 的 SSR 和静态生成能力

**实际应用**：
- 前端：React + Next.js 14 + Tailwind CSS + Framer Motion
- 后端：Next.js API Routes + Nodemailer
- 部署：Azure App Service

### 2. 项目架构设计

**核心原则**：
- **分层架构**：清晰分离前端展示、后端逻辑和数据处理
- **模块化设计**：将功能拆分为独立模块，提高代码可维护性
- **统一规范**：建立一致的代码风格和命名规范
- **可测试性**：设计便于单元测试和集成测试的架构

**实际应用**：
- 前端：组件化设计，页面与组件分离
- 后端：API 路由独立管理，逻辑清晰
- 国际化：统一的翻译文件管理

## 二、开发流程与协作

### 1. 开发流程管理

**核心原则**：
- **版本控制**：使用 Git 进行版本管理，建立清晰的分支策略
- **自动化部署**：配置 CI/CD 流水线，实现代码推送后自动构建和部署
- **持续集成**：及时发现和解决编译错误，保证代码质量
- **代码审查**：建立代码审查机制，确保代码质量和一致性

**实际应用**：
- GitHub 仓库管理，main 分支作为主分支
- GitHub Actions 自动部署到 Azure App Service
- 提交前运行类型检查和构建测试

**核心原则**：
- **系统性排查**：遇到问题时，从整体系统角度分析，而非局部调整
- **错误日志分析**：重视错误日志，快速定位问题根源
- **测试驱动**：通过测试验证修复效果，确保问题彻底解决
- **文档记录**：记录问题和解决方案，形成知识库

**实际应用**：
- 解决 Nodemailer 方法名拼写错误
- 修复 TypeScript 类型错误
- 解决 GitHub Actions 部署配置问题
- 优化多语言内容管理和更新流程

### 3. 协作与沟通

**核心原则**：
- **清晰的任务分配**：明确每个开发阶段的目标和责任人
- **及时的进度反馈**：定期沟通项目进展，及时调整计划
- **知识共享**：建立项目文档，确保团队成员了解整体架构
- **风险预警**：识别潜在风险并提前应对

**实际应用**：
- 分阶段完成前端、后端和部署任务
- 及时解决遇到的技术问题
- 建立完整的项目文档体系

## 三、部署与运维

### 1. 云平台选择

**核心原则**：
- **服务匹配**：根据项目需求选择合适的云服务（如 Azure App Service）
- **环境一致性**：确保开发、测试和生产环境的一致性
