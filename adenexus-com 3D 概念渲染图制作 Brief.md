## 1. 项目背景

ESGentics 是一家注册于新加坡的智能建造科技公司，面向东南亚、北美、欧洲、中东市场提供 AI 驱动的 A-Z 交钥匙工程服务。由于公司处于早期阶段，暂无可公开的实拍项目案例，需通过 **高质量 3D 概念渲染图** 展示技术能力与品牌调性。

### 1.1 核心目标

| 目标 | 说明 |
|------|------|
| 🎯 建立专业信任 | 通过工程级渲染质量，传递"欧洲标准 + 新加坡枢纽"的品牌形象 |
| 🎯 可视化 AI 能力 | 将抽象的"AI 算法 + 智能体 + 硬件"转化为可感知的视觉语言 |
| 🎯 支撑 ESG 叙事 | 在场景中自然融入可持续元素（太阳能、绿色屋顶、碳监测界面） |
| 🎯 规避地域风险 | 场景设定符合目标市场，**严格排除中国大陆关联元素** |

### 1.2 核心要求 ✅ / ❌

| 要求类型 | 具体内容 |
|----------|----------|
| ✅ 风格 | 欧式极简 + 工程美学 + 未来科技感，参考 Foster + Partners / BIG 概念图 |
| ✅ 内容 | 智能建造全流程、AI 算法可视化、ESG 可持续元素、NVIDIA 硬件暗示 |
| ✅ 区域 | 场景设定需符合 SE Asia/NA/EU/ME 建筑规范与环境特征 |

## 3. 视觉风格规范

### 3.1 色彩应用指南

```css
/* 品牌色在渲染图中的使用比例与场景 */
:root {
    --primary-red: #ed4f18;    /* 高光/交互节点/CTA 元素/预警信息，占比 ≤10% */
    --deep-blue: #283948;      /* 主背景/阴影/科技元素/界面底色，占比 ~40% */
    --light-gray: #f8f8f8;     /* 建筑主体/留白/界面卡片，占比 ~30% */
    --accent-green: #2ecc71;   /* ESG 可持续元素（可选），占比 ≤5% */
    --neutral-white: #ffffff;  /* 高光/文字/界面前景 */
}
```

| 色彩 | 使用场景 | 禁止场景 |
|------|----------|----------|
| `#ed4f18` (红) | AI 决策节点、数据流高亮、CTA 按钮、预警提示 | 大面积背景、建筑主体材质 |
| `#283948` (深蓝) | 夜空背景、界面深色模式、阴影、科技光晕 | 文字颜色 (需用白色替代) |
| `#f8f8f8` (浅灰) | 建筑外墙、界面卡片、留白区域 | 深色背景上的文字 |

### 3.2 光影与材质规范

| 元素 | 要求 | 参考关键词 |
|------|------|------------|
| **建筑材质** | 玻璃幕墙（反射天空，IOR 1.5）、混凝土（细腻 PBR 纹理）、金属框架（哑光银，roughness 0.3） | "archviz PBR materials" |
| **AI 可视化** | 半透明 holographic 界面（opacity 0.7, blur 10px），蓝色光晕 (`#283948` + glow 强度 0.5)，数据流用红色 (`#ed4f18`) 脉冲动画 | "holographic UI render", "data flow visualization" |
| **环境光** | 自然光为主：欧洲场景用柔和漫射光 (HDRI: overcast)，中东场景用强烈直射光 (HDRI: desert noon)，避免过度戏剧化 | "architectural lighting HDRI" |
| **人物元素** | 如需出现，必须为国际化多元面孔 (3-4 种肤色)，着装专业（工程服/商务装），姿态自然，**避免单一地域特征** | "diverse professional team 3d" |

### 3.3 禁止内容清单 ❌ (严格执行)

- [ ] 任何中文标识、招牌、文字、水印
- [ ] 中国地标（东方明珠、故宫、长城、广州塔等）或典型中式建筑风格（飞檐、斗拱、红墙）
- [ ] 中国大陆车牌（蓝底白字）、国旗、地图轮廓、行政区划
- [ ] 微信、支付宝、微博、抖音、百度等中国平台图标或界面元素
- [ ] 单一东亚面孔人群（需体现国际化团队：欧洲/中东/东南亚/非洲面孔混合）
- [ ] 服务器 IP 归属地、CDN 节点等暗示中国大陆基础设施的元数据

## 4. 技术交付要求

### 4.1 文件格式与分辨率

| 用途 | 格式 | 分辨率 | 色彩模式 | 压缩要求 | 备注 |
|------|------|--------|----------|----------|------|
| **Web 展示** | PNG-24 | 1920x1080 / 1600x900 | sRGB | TinyPNG 优化，单张 ≤500KB | 用于网站直接加载 |
| **Retina 屏** | PNG-24 | @2x (3840x2160) | sRGB | 可选压缩，单张 ≤2MB | 用于高清设备自适应 |
| **印刷/存档** | TIFF | 300 DPI, CMYK | CMYK | 无损 | 备用，非必需 |
| **源文件** | .blend / .max / .mb | - | - | - | 如需后续修改则提供，额外计费 |
| **动效备选** | MP4 (H.264) | 1920x1080, 5s 循环 | sRGB | ≤5MB | 仅 R01 可选，需提前确认 |

### 4.2 命名规范

```
esgentics_render_[场景编号]_[用途]_[分辨率]_[版本].[格式]

- esgentics_render_R01_hero_web_1920x1080_v1.png

## 6. 验收检查清单 (QA for Renders)

### 6.1 合规性检查 (必须 100% 通过)

- [ ] 所有渲染图无中文/中国地标/中式建筑元素/中国平台图标
- [ ] 人物面孔国际化多元，无单一地域特征
- [ ] 地图/节点标签仅包含 SG/EU/NA/ME，无 CN/China 字样
- [ ] 文件名、元数据、EXIF 中无中国大陆关联信息

### 6.2 品牌规范检查

- [ ] 品牌色 `#ed4f18` / `#283948` / `#f8f8f8` 应用准确（取色器验证，误差 ≤5%）
- [ ] AI 可视化元素清晰可辨，不过度拥挤，holographic 效果符合参考
- [ ] NVIDIA 相关元素（如芯片图标）符合官方品牌指南（如使用）

### 6.3 技术规格检查

- [ ] 文件格式、分辨率、命名符合 4.1-4.2 规范
- [ ] Web 版本经 TinyPNG/Squoosh 优化，单张 ≤500KB
- [ ] 加载测试：4G 网络下首屏图片加载 <1s (Lighthouse 验证)
- [ ] 每张图预留 "*Conceptual Simulation*" 标注位置（开发时添加）

### 6.4 视觉质量检查

- [ ] 构图平衡，主体突出，符合欧式极简美学
- [ ] 光影自然，材质 PBR 真实，无明显渲染噪点/穿模
- [ ] 色彩对比度 ≥ 4.5:1 (WCAG AA)，确保可读性
