# 企业级网站开发知识沉淀

## 核心方法论：AI 驱动 + 内容优先 + 技术规范

### 1. AI 辅助规范生成
- **输入**：企业 Branding 标准文档
- **处理**：AI 转换为技术规范（Tailwind 配置、开发约定、审查规则）
- **验证**：开发者审核 + 迭代优化

### 2. 内容驱动设计
- **策略**：收集内容 → 结构化内容 → 设计框架 → 填充内容
- **原则**：内容决定形式，而不是形式决定内容
- **实践**：多语言 JSON 管理、内容分层结构

### 3. 规范化技术栈
- **选型**：Next.js 14 + React 18 + Tailwind CSS 3 + TypeScript
- **边界**：明确技术栈边界，禁止随意引入新技术
- **组件化**：统一组件库（Button/Card/Section/Motion）

---

## 可复用蓝本

### A. 前端开发约定（Markdown）
- 颜色使用规范（只允许 tokens）
- 字体使用规范（title/body/cn）
- 布局规范（container-content）
- 组件使用规范（禁止重复造轮子）

### B. 代码审查规则（CheckList）
- 技术栈边界检查
- 样式系统检查
- 组件使用检查
- 国际化检查

### C. 项目结构模板
```
app/                    # 页面与路由
components/
  ├── layout/           # 布局组件
  ├── sections/         # 页面区块
  ├── ui/               # 通用 UI 组件
  └── motion/           # 动画组件
locales/                # 国际化文件
lib/                    # 工具函数
```

---

## 三大成功要素

1. **AI 读取 Branding → 输出开发约定**：规范项目边界，减少人为决策，提高代码一致性

2. **网站内容驱动设计**：内容决定形式，避免空洞框架，提升用户体验

3. **先进且规范的技术栈**：提供可扩展架构，降低技术债务，支持快速迭代

---

## 三大避免陷阱

1. ❌ **避免硬编码**：颜色/布局/文案必须使用规范
2. ❌ **避免重复造轮子**：必须使用统一组件
3. ❌ **避免过度设计**：保持简洁优于复杂

---

## 一句话总结

> **"AI 驱动 + 内容优先 + 技术栈规范 = 可复制的企业级网站开发蓝本"**

这个公式让新项目可以从经验开始，而不是从零开始，减少决策成本，提高开发效率。
